在树莓派上部署Openwrt

这个五一，我想在家里弄个各种设备都能用的梯子……

打算按照mlapp帖子里的固件和方法整，当然，有关树（Li）莓（nu）派（x）的任何东西都不会那么一帆风顺的……

使用macOS向SD卡写镜像

失败：用balenaEtcher写镜像直接报错，不管是lean官方的，还是mlapp提供的都说“镜像损坏”。

失败：用macOS的dd强行写，虽然能写进去，但返回值是错误的（bash返回错误，红箭头），插入树莓派启动失败：

网上搜了一下，推荐用Windows下的分区工具或者Linux下的dd。

用虚拟机向SD卡写镜像

用以前在virtualbox建好的里的win7，不论怎么设置苹果内置的sdcard reader（虚拟机里显示为一个USB设备，不论mount状态、如何重启虚拟机都不行），都报设备busy的错误：

查看sdcard：

或是卸载sdcard：

查帖子说是因为后来的MBP不把读卡器暴露为USB设备了，按这个帖子做：

  1. 插入SDcard，在终端里用mount查看分区，确定设备名称。在我的电脑上，显示为“NO NAME”的卷就是SD卡，其对应的设备叫做/dev/disk2s1，而这个s1表示/dev/disk2上的第一个分区，因此设备名其实应该叫做/dev/disk2：

  2. 打开电脑上“Disk Utility.app”，也就是“磁盘工具.app”，找到SD卡然后卸载它，要注意是卸载，而不是推出。另外，可能在做完下一步后还需要在卸载一次，因为macOS有可能自动把SD卡装载回来。

  3. 使用下面的命令，要注意的是，-rawdisk后面跟的参数是/dev/disk2，不是/dev/disk2s1。另外，/Users/zealot/VirtualBox\ VMs/WIN7/sdcard.vmdk这个路径写哪都行，图方便我直接放在我的WIN7虚拟机文件夹下了。：

  4. 重新配置虚拟机：

    1. 确定虚拟机已关闭；
    2. 打开“设置”->“存储”，点控制器旁边的“添加虚拟磁盘”，并选择“使用现有的虚拟磁盘”；
    3. 点击“注册”，选择上一步中生成的/Users/zealot/VirtualBox\ VMs/WIN7/sdcard.vmdk；此处如果报权限错误，我试了chmod和chown改.vmdk和disk2s1的权限是没用的，我也试了关掉macOS的SIP同样是没用的，必须命令行管理员权限运行sudo VirtualBox才可以。真的不太懂系统权限问题[捂脸]。
    4. 双击列表中的sdcard.vmdk，然后就会返回“存储”界面并看到这个设备。
    5. 启动虚拟机，看到该设备显示为一个本地磁盘。

  5. 这一套做下来，我们的SD卡在win7里就挂载为了一个ATA设备，也就是硬盘，很不幸的是“Win32 Disk Imager”是不认硬盘的。于是，把上一步的2修改一下：打开“设置”->“存储”，点控下方的“添加存储控制器”，选择菜单项“添加USB控制器”，这时候列表里多出来一个“USB控制器”，然后再按照上一步中2、3操作添加虚拟磁盘，再选上为“热插拔|固态驱动器”，这样就真的把SD卡伪装成USB设备了。此时Win7里出现两块硬盘，一个是ATA设备，一个是USB设备。再打开“Win32 Disk Imager”时，就可以选择这个盘了：

开机搜索名为“openwrt”的无线SSID，连接，进入http://192.168.1.1，用“root/password”登录配置界面。

等Openwrt配置

先让自己能上网

我家用的TP-LINK的猫，LAN口接树莓派，而这个TP-LINK默认是在192.168.1.1上的，所以先在openwrt配置界面中“网络”->“接口”里改个IP，默认网关写在192.168.1.1上，然后写上俩TP-LINK里用的DNS就可以访问互联网了。

配置SS

先把本机的公钥先传到openwrt上，方便ssh登录，然后直接root登录吧，懒得弄用户了。

如果vim时报错遇到"E437: terminal capability "cm" required"，需要执行一下export TERM=xterm。

继续，在openwrt管理页面中配置“ShadowSocksR Plus+”->“服务器节点”，添加一个类型为“ShadowSocksR”的服务器就可以了，我的情况，主机名、端口、密码、加密方式填上，其他默认就可以直接联通了。

github免密push

临时想到的，最近是不是要更新博客，push的时候总是让我输入用户名密码，很烦。因为我的github是双因子认证的，总要输入用户名，然后复制粘贴那个巨长的push key。

github特定仓库的免密push设置也很简单，最关键的是注意协议。

本机git仓库

在仓库中使用git config -e命令修改配置文件，只有使用git协议是才可以免密，https不行：

如果是hexo之类的博客，cli工具自动集成push功能，那么修改其配置文件的repo目标即可，比如我用的hexo，那么修改_config.yml即可：

github配置

可以在你的某个特定github-repo下的“settings”->“deploy key”里写一个公钥，然后允许写权限。不过一个库会占一个key，下一个库就不能再用这个key了。

也可以直接在github账号配置的“settings”->“SSH and GPG keys”里写一个SSH公钥，这个方法更简单，推荐。
